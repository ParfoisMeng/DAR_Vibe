## 1. 四类游戏特性分析

在最终设计前，先厘清各类型游戏的**本质乐趣**，避免错误借鉴。

### 自动化/放置游戏

**本质**：构建一套系统 → 让它自动运行 → 看结果 → 调优再优化。

乐趣来自**优化**，不是等待。"放置"是战斗/推进的**执行方式**，不是时间墙。

| 应保留 | 应抛弃 |
|-------|-------|
| 战斗自动结算，无需手动操作 | 「等待30分钟升一级」这类时间墙 |
| 系统自动产出结果，玩家做决策 | 需要长时间放置才能推进的核心循环 |

### 肉鸽游戏

**本质**：有限局内决策积累 + 协同效果发现感 + 每局不同的新鲜感。

每次Run = 独立的Build构建之旅。元成长保证「死了不白死」。

| 应保留 | 应抛弃 |
|-------|-------|
| 局内「N选1」决策节点（Build的核心时刻）| 需要大量时间的境界等待成长体系 |
| 协同效果/Build发现感，永久元成长 | 与10-20分钟单局不兼容的任何长流程 |

### 暗黑刷子

**本质**：对「理想Build」的永恒追求。词缀系统创造近乎无限的Build可能性。

核心动机：「我的目标Build缺词缀A+B，刚掉了一件有A的，离目标又近了一步！」

| 应保留 | 应抛弃 |
|-------|-------|
| 词缀装备、Build多样性、稀有掉落惊喜感 | 繁琐的装备管理——短Run里决策必须轻量 |
| 炼化/改造系统（Run间进行）| 每局内大量时间用于装备对比管理 |

### 文字MUD

**本质**：文字作为界面提供「质感」和「代入感」——廉价但有效的沉浸感来源。

关键：文字日志让自动战斗有画面感；简短选项事件让玩家感觉「我在一个世界里」。

| 应保留 | 应抛弃 |
|-------|-------|
| 3~5行战斗日志（简洁，有画面感）| 长篇叙事、复杂分支剧情 |
| 简短2~3选项的随机事件 | 时效事件队列、复杂世界状态系统 |

## 2. 核心设计定论

### 关键矛盾及解法

**矛盾**：「放置游戏」× 「10-20分钟单局」天然冲突。

**解法**：「放置」重新定义为战斗的**执行方式**，而不是时间的消耗方式。

```
❌ 旧理解：挂机等待修炼值满 → 升级 → 再等待   （一局需要数小时）
✅ 新理解：战斗自动秒结算   → 掉落  → 立刻决策  （一局10~20分钟）
```

### 三核融合最终定位

| 层次 | 来源类型 | 本作实现 | 比重 |
|-----|---------|---------|-----|
| **主干**：Build构建决策流 | 肉鸽 | 境界节点3选1功法，8~10次/局 | ★★★★★ |
| **执行**：自动化战斗 | 放置/自动战斗 | 战斗15~30秒自动结算，玩家不操作 | ★★★★ |
| **深度**：词缀装备追求 | 暗黑刷子 | 掉落词缀装备，Run间炼化，多局Hunt | ★★★★ |
| **质感**：文字叙事氛围 | MUD | 战斗日志+简短随机事件，纯增益 | ★★ |

## 3. 单局节奏设计

### 3.1 时间分布（目标：15分钟/局）

```
Run 开始
│
├─ 灵根选择（20秒）
│
├─ [区域 1] ─ 3场自动战斗（约90秒）
│              ↓ 每场战斗后：文字日志（5秒浏览）
│              ↓ 遭遇事件节点（随机，30秒）
│              ↓ 【境界节点】3选1功法（30秒决策）
│
├─ [区域 2] ─ 3场自动战斗（约90秒）
│              ↓ 精英怪（强化战斗，30秒）
│              ↓ 装备掉落确认（10秒）
│              ↓ 【境界节点】3选1功法（30秒决策）
│
├─ [Boss 1] ─ Boss自动战斗（30~60秒，可点主动技能）
│              ↓ Boss掉落（必出装备）
│
├─ [区域 3~4] ─ 重复，难度递增
│
├─ [最终Boss] ─ 60~90秒，多阶段
│
└─ 结算（30秒）→ 道韵结算 → 下一局

总计：约 12~15 分钟（含决策时间）
```

### 3.2 节奏感设计原则

1. **「快→决策→快→决策」交替节奏**：战斗快速自动结算，决策节点给玩家停下来思考的时间
2. **每场战斗都产出反馈**：掉落、修炼值、文字日志——不能有「什么都没发生」的空白战斗
3. **死亡要快**：挑战失败立刻结算，不拖时间，尊重玩家时间
4. **Boss有仪式感**：特殊BGM/文字描述，和普通战斗明显区别

### 3.3 区域结构

每局有 **4~5个区域**，每个区域包含：

```
区域结构（每个约3分钟）：

  3场普通战斗（自动）
  + 0~1个文字事件节点（随机触发）
  + 1个装备/功法商铺节点（可选进入）
  + 1个精英战斗节点（可选挑战，风险更高收益更好）
  + 1个境界节点（必触发，3选1功法）
  ↓
  Boss节点
```

### 3.4 道途分叉（每层Boss后触发）

通过Boss后，玩家需在**两条路线**中选择进入下一区域，让「选择道途」字面成立：

```
━━━ 道途分叉：前方有两条路 ━━━

  [北路] 古战场 · 险象环生
         ─ 精英怪密度更高，掉落品质提升
         ─ 本路线专属功法节点：「战意磅礴」「血战不退」
         ─ 风险：可能遭遇双精英

  [南路] 幽谷秘境 · 灵气充沛
         ─ 事件节点增多，商铺出现概率+50%
         ─ 本路线专属功法节点：「灵气感悟」「洞天福地」
         ─ 风险：较少，战斗难度偏低
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**设计意图**：
- 两条路线的专属功法节点让「路线选择」与Build方向挂钩
- 玩家在追求特定协同时会有明确的路线偏好
- 路线选择每局固定2次（第1层Boss后、第2层Boss后）

## 4. Build 构建系统（肉鸽精华）

这是整个游戏设计的**核心**，直接决定游戏性强度。

### 4.1 灵根（Run初始选择）

灵根决定初始属性偏向和**功法池权重**，相当于肉鸽的职业：

| 灵根 | 初始属性侧重 | 专属机制 | 典型Build方向 |
|-----|-----------|---------|-------------|
| 纯阳剑灵根 | 攻击+暴击 | 【剑意】叠层 | 暴击连击流 |
| 炎华火灵根 | 爆发+攻速 | 【灼烧】DoT | 点燃灼烧流 |
| 玄冥水灵根 | 速度+控制 | 【寒气】减速 | 控制削减流 |
| 厚土炼体根 | 防御+反伤 | 【金刚】护盾 | 防反金身流 |
| 五行混元根 | 均衡 | 【五行循环】协同 | 万能高上限流 |

### 4.2 功法节点系统（核心决策）

每次到达境界节点，随机展示**3个功法节点供选择1个**，加入当前Run的功法树。

**每局触发境界节点次数**：8~10次（每次都是重要决策）

**选择界面必须显示「当前功法树简览」**（纯游戏性审查 Q1 补入）：

```
境界节点选择界面布局：

  ┌─ 当前功法树 ──────────────────┐
  │ 「剑意积累」→「趁虚而入」     │  ← 已选节点，让玩家感受到累积感
  │  相关协同：3个（已触发：1个）  │  ← 提示有协同潜力，不展示内容
  └──────────────────────────────┘

  请选择下一个节点（3选1）：
  [A] 剑意外放   [B] 凌空踏虚   [C] 百炼金身
```

> 这个设计让玩家在每次决策时能「回望」已有的Build，产生「这个选择是否和上面的有关联」的期待感，是心流触发的核心条件。

```
【主动技能】
  有冷却时间，战斗中可手动激活（Boss战主要干预方式）
  示例：「斩魄剑」—— 造成350%攻击伤害，忽视护盾，冷却90秒

【被动增益】
  永久生效，无条件触发
  示例：「剑意积累」—— 每次攻击叠加1层剑意（上限100层），暴击率随层数提升

【条件触发】
  满足特定条件时自动激活，是协同效果的主要来源
  示例：「趁虚而入」—— 攻击减速状态下的敌人时，伤害+40%
```

### 4.3 协同效果（Build发现感的核心）

特定节点组合同时存在时，解锁**隐藏协同效果**。

#### 协同分层（解决触发概率问题）

协同效果按触发难度分三层，**部分普通协同改为单节点触发**（评审会 A19 修正）：

| 层级 | 触发条件 | 数量 | 触发期望频率 | 效果强度 |
|-----|---------|------|-----------|--------|
| 普通协同·A型 | 含节点A即触发（含节点B时效果加强）| 8个 | 每局期望 1.1次 | 20~40% 增益 |
| 普通协同·B型 | 2个特定节点组合 | 7个 | 每局期望 0.7次 | 40~60% 增益 |
| 稀有协同 | 3个节点组合 | 25个 | 每局期望 0.5次 | 60~100% 增益 |
| 史诗协同 | 3节点+特定灵根 | 10个 | 每10局约1次 | 彻底改变战斗逻辑 |

> **期望采用总计：** (8×个单节点期望) + (7×0.1) + ... ≈ 1.8次/局，保证每局大概率触发 1~2 次普通协同

```
普通协同·A型 示例（单节点触发）：
  「剑意积累」（必要）+ 「趁虚而入」（可选，加强）
  → 基础：剑意层数叠加速度+50%
  → 加强：还额外，趁虚而入的伤害加成也+30%

普通协同·B型 示例（双节点触发）：
  「剑意积累」 + 「剑意外放」
  → 剑意100层时自动释放全场剑气，造成200%伤害
```

#### 协同触发仪式感

协同触发时，界面展示专属通知（3秒后消失）：

```
╔══════════════════════════════╗
║  【天机触发 · 普通】          ║
║  剑意与冰封，相生相克         ║
║                              ║
║  「减速状态下，剑意叠加×2」   ║
╚══════════════════════════════╝
```

史诗协同触发时有更强的视觉/文字效果，明确告知玩家这局非同凡响。

#### 协同发现记录

玩家可在主界面查看「天机图鉴」——已发现的协同效果记录：
- 未发现的协同显示为「???」（只显示槽位数量提示）
- 已发现的协同显示完整描述和触发条件
- 发现进度可作为老玩家的收集目标

**设计原则**：
- 协同效果在功法选择界面**不提示**（天机图鉴里可查）
- 普通协同保证玩家新手期就能体验，史诗协同作为深度目标
- 总计约 50 个协同效果（15 普通 + 25 稀有 + 10 史诗）

### 4.4 功法节点数量规划

```
Phase 0 原型：20个节点，5个协同效果
Phase 1：    60个节点，20个协同效果
Phase 2：    120个节点，50个协同效果
```

## 5. 战斗系统（自动结算）

### 5.1 战斗模型

战斗完全自动，基于属性对比秒算结果：

```
战斗结算公式（简化）：

  玩家DPS = 攻击力 × 攻速 × 暴击系数 × 技能触发系数
  敌人TPS = 敌人攻击 × 敌人攻速 × (1 - 玩家减伤)

  击杀时间 = 敌人HP / 玩家DPS
  死亡时间 = 玩家HP / 敌人TPS

  若 击杀时间 < 死亡时间 → 胜利，结算掉落
  否则 → 失败，HP扣除（普通怪）或直接死亡（Boss）
```

普通战斗**不消耗玩家HP**（被动技能可增加容错），只有精英怪和Boss才造成实质伤害。
这样保证战斗快速推进，不会因为HP管理拖慢节奏。

### 5.2 战斗日志（实时滚动，解决等待感）

战斗自动进行时，日志**实时逐行滚动显示**（不是战后一次性呈现），给玩家在决策间隙有内容可看：

```
━━━ 战斗 · 北荒荒野 · 第二区域 ━━━
  遭遇：「荒野煞魂」×3

  > 你御剑而上，剑气横扫，煞魂群受到 847 点伤害
  > 剑意积累至 23 层，暴击率上升至 28%               ← 实时显示状态变化
  > 「荒野煞魂·首领」发狂，攻击力提升
  > 你护体真气激活，抵消 120 点伤害
  > 会心一击！「首领」受到 2,340 点伤害              ← 暴击高亮
  > 「荒野煞魂」全部溃散

  ✦ 获得：[珍品] 枯骨腰带（攻击+18，暴击+3%）
  ✦ 剑意 +3层（当前：23层）
━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**实时日志设计规则**：
- 每条日志以 `>` 开头，以 100~200ms 间隔逐行出现
- 特殊事件高亮（会心一击、协同触发、防反）
- 战斗结束后日志保持可滚动查看
- 战斗进行时，玩家可查看当前状态（HP、叠层数）但不能做其他操作（保持「等待感」最小化）

### 5.3 Boss战：「天机时刻」机制（全新设计）

Boss战**彻底区别于普通战斗**——不再是「被动看日志偶尔点技能」。

#### 核心机制：天机时刻

Boss战自动进行时，在特定触发条件下，**游戏完全暂停**，展示一个需要玩家做决策的关键时刻：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  ⚡ 天机时刻 · 苍狼王的怒吼
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  「苍狼王」嗷叫长嚎，全身乍现血光。
  它正在蓄积必杀之力——下一击将造成你当前生命的 150% 伤害。

  你感到天机流转，时间凝固。应对之策，只在此刻。

  ─────────────────────────────────
  [A] 护体真气全开
      → 本次伤害减至 20%，消耗当前护盾全部值

  [B] 趁虚全力突进（需：剑意 ≥ 50 层）✓ 当前：67层
      → 打断蓄力，造成 800% 攻击伤害
        有 35% 概率直接击杀（Boss HP < 30%时）

  [C] 凌空闪退，以待时机
      → 完全回避本次攻击，但「剑气流」协同效果中断至本战结束
  ─────────────────────────────────
  ※ 无时间限制，慎重选择
```

#### 天机时刻触发条件

| 触发时机 | 每场Boss次数 | 说明 |
|--------|------------|------|
| **Boss战开始时（开场仪式）** | **1次（必须）** | **无论Build强弱都必触发，设置本场Boss的特殊规则（纯游戏性审查 Q4 补入）** |
| Boss HP 降至 75% | 1次 | 警告阶段，相对温和 |
| Boss HP 降至 50% → 进入第二阶段 | 1次 | 阶段变化，规则改变 |
| Boss HP 降至 25% → 「垂死反扑」 | 1次 | 最危险时刻，最强选项 |
| 玩家 HP 降至 30% 以下 | 触发1次 | 仅在玩家濒死时出现 |

**每场Boss战天机时刻：3~5次（1次开场 + 2~3次血量阈值）**

> **开场天机时刻说明**：Boss战第一秒触发，内容是「设置本场Boss的特殊规则」（如「Boss本场拥有魔化护盾，每次受击恢复5%血量」vs「Boss本场失去感知，攻击频率-50%但每次造成200%伤害」）。这个天机时刻不是「防御性的」，而是「主动设置战局的」，让玩家一进入Boss战就感到掌控感。

#### 选项设计原则

1. **选项可用性由Build决定**：「需剑意≥50层」的选项只在玩家有此机制时出现
2. **永远有一个保守选项**（代价是资源消耗或暂时削弱）
3. **最优选项因Build不同而不同**——「剑气流」和「金身流」的最优解完全不同
4. **后果立刻呈现**：选择后日志实时显示结果
5. **每种灵根/核心机制必须有专属可用选项**（纯游戏性审查 Q11 补入）：
   - 剑灵根专属选项：需要「剑意≥X层」的爆发型选项
   - 炼体灵根专属选项：需要「护甲值≥X」的防反型选项
   - 法道灵根专属选项：需要「当前施法数≥X」的法力爆发型选项
   - 通用选项：每个天机时刻至少有1个无条件可选的选项（保守选项）
   > **设计底线**：「Build决定你有什么可选」是体验差异化的核心，哪怕玩家每局选同一灵根，面对同一Boss的天机时刻，可选的选项也应因为这局的Build路径而不同。

#### Boss第二阶段

Boss HP降至50%时触发，规则发生改变：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  ⚡ 天机时刻 · 苍狼王·化神之怒
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  「苍狼王」身形骤变，狼毛化为寒铁，双眼赤红如血。
  它进入了第二阶段——原有的剑意克制被它化解。

  它正在施展「魔化狂怒」：你的剑意积累效率下降 50%。

  [A] 接受削弱，继续以剑气硬推
      → 正常推进，但剑意Build效率减半

  [B] 临时转换「炼体」策略（需：功法树含炼体节点）✓
      → 本战切换为炼体防反模式，无视剑意削弱

  [C] 使用「封印符」（需：持有封印符×1，商铺购买）
      → 抵消本次阶段变化效果
```

> 第二阶段的天机时刻会**针对玩家Build的弱点**出牌——这是重开动力和Build多样性的重要来源。

### 5.4 核心属性

```
攻击端：攻击力、攻速、暴击率、暴击伤害、穿透
防御端：生命上限、护盾值、减伤率、闪避率
特殊端：掉落幸运（词缀质量）、天命值（事件权重）
```

## 6. 装备与词缀系统（暗黑层）

### 6.1 装备槽位（6槽，精简版）

```
[ 武器 ] [ 护甲 ] [ 饰品1 ] [ 饰品2 ] [ 戒指 ] [ 特殊 ]
```

> 精简为6槽，减少管理负担，每件都有决策重量。

### 6.2 品质体系

| 品质 | 词缀数 | 掉落来源 |
|-----|-------|---------|
| 凡品 | 0 | 普通怪，忽略 |
| 灵品 | 1 | 普通怪，常见 |
| 珍品 | 2 | 精英怪，常见 |
| 极品 | 3 | 精英怪/Boss，较稀 |
| 仙品 | 4 | Boss必掉1件，很稀 |
| 混沌品 | 3~5+特殊规则 | 秘境专属，极稀 |

### 6.3 词缀分层设计

词缀分三层，层次越高越稀少、Build价值越大：

**第一层：基础词缀**（任何品质都可出现）
- `[锋利]` 攻击力 +X
- `[强韧]` 最大生命 +X
- `[神速]` 攻速 +X%
- `[敏锐]` 暴击率 +X%

**第二层：触发词缀**（珍品以上）
- `[嗜血]` 击杀时回复 X% 最大生命
- `[雷引]` 攻击有X%概率附带雷击（额外Y%伤害）
- `[寒气侵体]` 命中减速敌人时额外暴击率+X%
- `[以血饲剑]` 生命低于30%时攻击力+X%

**第三层：协同词缀**（极品以上，Build定义级）
- `[剑意涌现]` 每20层剑意，攻击力+X（上限Y）
- `[灼烧蔓延]` 灼烧伤害有X%概率蔓延至相邻敌人
- `[金刚护体·主动]` 激活主动技能时获得相当于最大生命X%的护盾

**稀有专属词缀**（仙品/混沌品专属，全局唯一）
- 独特机制，不可通过炼化获得，只能靠掉落
- 示例：`[万剑归宗]` 所有主动技能冷却-60%，但每次激活消耗X点生命

### 6.4 装备决策设计（Run内）

Run内的装备决策要极度**轻量化**：

```
掉落装备时，界面展示：

  ┌─────────────────────────────────┐
  │  [珍品] 寒铁腰带                │
  │  攻击力 +22  暴击率 +5%         │
  │  [寒气侵体] 减速敌人时暴击+8%   │
  │                                 │
  │  当前装备：粗布腰带             │
  │  攻击力 +12                     │
  │                                 │
  │  [装备] [保留备用] [分解→材料]  │
  └─────────────────────────────────┘
```

三个操作，5秒决策，不拖节奏。

### 6.5 Run间：炼化系统

Run结算后，在「炼丹炉」界面对收集的装备进行炼化（无时间压力）：

| 操作 | 消耗 | 效果 |
|-----|------|------|
| 重铸词缀 | 灵石 ×50 | 随机替换一个词缀 |
| 淬炼强化 | 对应材料 | 提升一个词缀的数值 |
| 融合 | 两件同槽位装备 | 各取1个词缀合并（销毁原件）|

### 6.6 执念系统（Hunt目标标记）

玩家可以标记最多 **3个目标词缀** 作为「执念」，解决「不知道在Hunt什么」的问题：

```
执念设置界面：

  我的执念（当前 2/3）：
  ├─ [剑意涌现]    —— 协同词缀，正在Hunt中...
  └─ [万剑归宗]    —— 稀有专属词缀，极稀...

  [+ 添加执念词缀]
```

**执念的效果**：
- 装备掉落时，若含有执念词缀，掉落框**高亮 + 特殊提示**
- 战斗结算后的「收获汇总」中，执念词缀装备优先显示
- 执念词缀装备不会被系统自动分解（防止误操作）

**执念软保底机制**（评审会 A9 虚点补入）：

防止「200局都没出」的挂败感，采用软保底而非硬保底：

| Hunt无效局数 | 被动触发 | 效果 |
|------------|-------|------|
| 连续 10局 | 秘境掉落概率微升 | 目标词缀出现概率 +10% |
| 连续 20局 | 商铺出现专属选项 | 可以用道韵直接购买该词缀装备 |
| 连续 30局 | 天机感应事件 | 触发一个专属事件，保证获得该词缀 |

> 软保底只影响概率和事件，不直接给予装备，保持稀有词缀的稀缺感

**执念发现引导**（新手向）：
- 首次触发某个协同效果后，游戏推荐将该协同的装备词缀加入执念
- 在天机图鉴查看已发现协同时，可直接点击「追求相关词缀」添加到执念

## 7. 文字质感层（MUD精华）

**定位**：文字是界面风格，不是游戏核心内容。目标是用最少的文字营造最大的代入感。

### 7.1 文字事件（每局2~4个）

每个事件：**2~3行描述 + 2~3个选项**，30秒内处理完毕：

```
━━━ 事件：古修遗迹 ━━━

  荒野中，你发现一处被荒草掩埋的石台。
  台上刻有残缺的功法，灵气稀薄但仍可感悟。

  [A] 坐下感悟  → 立刻获得随机功法节点碎片（可用于换取特定功法）
  [B] 带走石台  → 获得炼化材料 ×5
  [C] 不感兴趣  → 继续前进
━━━━━━━━━━━━━━━━━━
```

事件结果**立刻生效**，无动画，无等待。

### 7.2 战斗日志风格规范

- 每场战斗日志：3~6行，不超过10行
- 用动词带动感：「御剑横扫」「凌虚踏空」「剑意大盛」
- 数字有意义：伤害数字、叠层数、获得资源要清晰
- 特殊情况高亮：暴击、协同触发、防反等要有标识
- **日志需实时显示叠层数变化**（每次变化都输出，如「剑意：23→24→27（会心）→35（协同加成）」），让玩家在等待期感受到数值积累的动感

**协同触发日志语言规范**（纯游戏性审查 Q2 补入）：

> 协同触发的日志是全局「最重要的文字」，必须使用仙侠叙事语言，**严禁出现机制描述文字**。

| ❌ 禁止写法 | ✅ 正确写法 |
|-----------|----------|
| 「协同效果已激活，伤害+60%」 | 「天机触发·剑道宗师：两道之间，因果已定」 |
| 「剑意协同：触发，叠加效果生效」 | 「剑意与虚空相合，动中有静，招招皆是道」 |
| 「史诗协同·激活成功」 | 「天机大开·化剑为道：此局，已无人可敌」 |

### 7.3 界面文字风格

所有UI文字均使用仙侠语境：

| 通用游戏术语 | 本作文字 |
|-----------|--------|
| 攻击力 | 剑气（或「锋芒」）|
| 生命值 | 根基 |
| 护盾 | 护体真气 |
| 暴击 | 会心一击 |
| 装备 | 法宝 |
| 材料 | 灵材 |
| 金币/货币 | 灵石 |
| 永久货币 | 道韵 |

## 8. 肉鸽循环设计

### 8.1 Run结构与时间分布

```
[灵根选择] 20秒
    ↓
[区域1] 约3分钟
  └─ 战斗×3 + 事件×1（随机）+ 境界节点×1
    ↓
[区域2] 约3分钟
  └─ 战斗×3 + 精英×1 + 境界节点×1
    ↓
[Boss1] 约1分钟
    ↓
[区域3] 约3分钟（同上）
    ↓
[区域4] 约3分钟（难度提升）
    ↓
[最终Boss] 约1.5分钟
    ↓
[结算] 30秒

总计：~15分钟
```

### 8.2 Run内关键数字

| 项目 | 数量 |
|-----|------|
| 境界节点（Build决策）| 8~10次 |
| 随机事件 | 3~5个 |
| Boss战斗 | 2~3场 |
| 装备掉落机会 | 6~10次 |
| 协同效果触发（期望）| 1~2次 |

### 8.3 随机性分层

| 随机点 | 机制 |
|-------|------|
| 功法节点候选 | 每次3个随机（受灵根权重，防止完全无关选项）|
| 地图区域布局 | 节点连接随机，但结构模板固定（确保可玩性）|
| 事件内容 | 从事件库随机抽取（约50个事件的池）|
| 装备词缀 | 随机（受掉落幸运影响品质和词缀层次）|

### 8.3.1 每局底线保障规则（纯游戏性审查 Q3 补入）

防止「块Run」（所有随机结果全部平庸、没有任何了不起的局），每局强制保障最低质量线：

| 保障项 | 规则 | 原因 |
|--------|------|------|
| 境界节点底线 | 第2次境界节点**必须出现至少 1 个「相关协同≥1」的节点候选** | 确保每局少有一条可期待的协同追求线索 |
| 事件底线 | 每局第一个文字事件必须是「中等收益」以上的事件（禁止“全部选项都平庸”的事件作为第一个事件） | 首个事件必须给玩家一个「有收益的开局感」 |
| 协同底线 | 第一局强制保证触发至1个普通协同（序章节点池权重调整） | 除首局必须让玩家知道“这个机制存在” |

> **底线保障不是局内免死顽，而是确保每局都出现至少一个「候幸运女神」的局底机会。**

### 8.4 死亡设计

- **硬死**：Run结束，当前局功法树/临时状态清零
- **保留**：道韵值、已解锁的炼化材料存档、装备存入仓库（Run间共享）
- **死亡文字**：一句话结局描述，有代入感，不说教

**死亡结算界面设计**（评审会 A15 虚点补入）：

```
━━━ 历劫封印 ━━━

  "一剑切云，年少不知天高地厚。"

  其山 Build 展示：
  筑基→ 「剑意积累」 → 「趁虚而入」 → 「剑意外放」
  天机触发：「剑意与冰封，相生相克」（普通协同）
  达到：元婴期 / 死于：苍狼王（天机时刻2选错误）

  获得道韵：142

  ※ 若上次选择「护体真气」而非「趣险突进」，结果将完全不同。

  [弘光再起] [查看天机图鉴] [等待片刻]
━━━━━━━━━━━━━━━━━━━━
```

**设计原则**：结算界面要让玩家看到「如果上次选了……」就能改变结局，触发重开欲望而不是挫败感。

**死亡精确归因要求**（纯游戏性审查 Q8 补入）：

结算界面必须做到**精确指向最关键的一次天机时刻决策**，不能模糊归因为「数值不够」：

```
精确归因展示示例：

  「关键时刻回顾」
  
  ✗ 天机时刻2（Boss HP 50%）
    你选择了：[A] 护体真气全开（减伤，消耗全部护盾）
    
    当时你也可以选择：[B] 趁虚全力突进（需剑意≥50层 · 当时你有67层 ✓）
    → 若选[B]：造成800%伤害，有35%概率直接终结Boss
    
  → 条件满足，但你没有选。下次也许可以试试。
```

> 让死亡从「数值失败」变成「可审视的决策失败」，每次死亡都有一个清晰的「如果当时…」反思路径。

## 9. 永久成长系统

### 9.1 道韵传承树（三个方向）

```
【道途】── 扩展每局的选择空间
  - 解锁新灵根（购买后每局可选）
  - 开局额外获得1个功法节点（直接选）
  - 解锁起始特殊装备选项

【执念】── 提升刷宝效率（可叠加购买）
  - 掉落幸运 +5%（最多×10层）
  - 精英怪掉落率 +10%
  - 炼化材料消耗 -5%

【因缘】── 扩展事件池和协同发现
  - 解锁新的随机事件（增加事件池深度）
  - 特定门派声望起始+1（影响相关事件选项）
  - 天命值上限 +10（影响稀有事件概率）
```

### 9.2 仓库系统（跨Run共享）
**核心规则（玩家Q&A Q13 补入）：Run 结束时（无论通关还是死亡），本次Run获得的所有装备全部自动进入仓库。死亡不消耗装备。**

> 这与「暗黑 角色死亡装备不消失」是同一逻辑——让刺宝行为本身有积累感，玩家每局必有收获。
装备可以跨Run保存到仓库（上限20件），用于：
- 积累特定词缀装备，等待更好的融合时机
- 跨Run构建特定词缀集合

```
      ├── Equipment.vue     # 装备管理
      └── Forge.vue         # 炼化界面（Run间）
```

### 11.4 Run 断点续玩（评审会 A13 虚点补入）

解决 PWA 定位与 15 分钟单局的矛盾：

**实现方式**：玩家随时可关闭浏览器，下次打开自动恢复到上次离开时的状态（精确到当前区域）。

```
Run状态快照话存内容：
  - 当前区域位置
  - 当前功法树（已选节点列表）
  - 当前装备状态
  - 当前 HP / 叠层数
  - 正在进行中的战斗状态（若在战斗中离开，重新进入时重打该场战斗）

存储方式： localStorage 即可实现，Phase 0 必须实现
```

**意义**：「随时随地」的语义变为「随时开始、随时暂停、随时继续」，不需要连续 15 分钟。

### 11.5 存档安全方案（玩家Q&A Q9 补入）

> **localStorage被清除是Web平台最影响长期留存的风险**，必须分阶段解决。

| 阶段 | 存档方案 | 说明 |
|------|----------|------|
| Phase 0~1 | **导出存档码** | 将存档序列化为Base64字符串，玩家可手动复制保存，下次输入即可恢复；这是广泛使Web游戏的标准临时方案 |
| Phase 2 | **Supabase 云存档** | 免费层即可支敲早期用户规模，登录后自动同步存档，彻底解决存档丢失问题 |

**导出存档码 UI 设计**：
```
设置页面 「存档管理」：

  [导出存档码]  复制此符串并抖操保存
  [输入存档码]…

  导出示例： AABB-CCDD-2026XXXXXXXX...

  【注意】游览器清除缓存会导致存档丢失，建议定期导出存档码备份。
```

## 12. Phase 0 验证策略

Phase 0 的目标不是做一个完整游戏，而是**证伪或证实三个核心假设**，决定是否继续投入。

### 12.1 三大核心假设

| 假设 | 验证方法 | 通过标准 |
|-----|---------|---------|
| **H1：3选1功法节点的决策感足够** | 10次游玩，记录每次选择时的犹豫程度 | 80%的选择让玩家感到「有点难选」 |
| **H2：协同触发创造了想要的爽感** | 第一局保证触发1个普通协同（设计引导）| 玩家主动描述想找更多协同 |
| **H3：「天机时刻」让Boss战有参与感** | 对比「纯自动Boss」和「天机时刻Boss」| 玩家明确偏好天机时刻版本 |
| **H4：自动战斗不产生无聊感** | 测试者战斗期间的行为观察（是否拿手机/看其他页面）| 80%的战斗期间玩家视线持续留在屏幕上 |

> **H4 说明**（玩家Q&A Q1 补入）：自动战斗是本设计最大风险点。如果Phase 0中有玩家在战斗期间拿手机或切换标签页，需立即评估是否为普通战斗增加低频点击交互（如每场战斗1次「主动放技能」窗口）。

### 12.2 Phase 0 最小范围（2周）

**只做够验证假设的内容，不多一行代码：**

```
内容范围：
  ├─ 灵根：仅「纯阳剑灵根」（最直观）
  ├─ 区域：2个区域 + 1个Boss（最短闭环）
  ├─ 功法节点：20个（10个剑系核心 + 10个通用）
  ├─ 协同效果：5个（3普通 + 1稀有 + 1史诗）
  │              ※ 第1局必须触发1个普通协同（通过节点池概率设计保证）
  ├─ 装备：3槽（武器/护甲/戒指），15个词缀
  ├─ Boss：1个（含天机时刻×2）
  ├─ 文字事件：5个
  └─ 存档：localStorage，无永久传承（先不做）

技术：纯 HTML + Vanilla TypeScript，无框架
```

### 12.3 验证流程（2周计划）

```
Week 1（第1~7天）：
  Day 1~2：战斗自动结算引擎（DPS公式 + 日志生成）
  Day 3~4：功法节点系统（3选1 + 协同检测）
  Day 5~6：Run流程控制（区域推进 + 境界节点触发）
  Day 7：装备掉落 + 基础UI

Week 2（第8~14天）：
  Day 8~9：Boss战 + 天机时刻（2个触发点）
  Day 10~11：文字事件系统
  Day 12~13：调试 + 数值调整（确保单局10~15分钟）
  Day 14：内部游玩测试，收集反馈
```

### 12.4 验证通过标准（Go/No-Go 决策门）

游玩5~10局后，评估以下指标：

| 指标 | 通过（继续Phase 1）| 失败（重新设计）|
|-----|----------------|--------------|
| 单局时长 | 10~18分钟 | <8分钟或>20分钟 |
| 协同触发爽感 | 玩家主动提到想再找协同 | 玩家对协同触发无反应 |
| 功法选择质量 | 「有时候真的不知道选哪个」 | 「随便选都一样」 |
| 天机时刻 | 「Boss战比普通战斗紧张」 | 「感觉差不多」 |
| 重开欲望 | 死后马上想重开 | 觉得没意思就关掉 |

### 12.5 已知风险与预案

**风险：协同概率设计失准**
- 如果前3局都没触发协同 → 游戏最大卖点被掩盖
- **预案**：Phase 0阶段第一局强制保证剑意相关协同触发（通过节点池权重调整），后续恢复随机

**风险：天机时刻选项不够有意义**
- 玩家发现「总是选同一个选项最优」
- **预案**：准备两套天机时刻选项（高攻Build方案 vs 高防Build方案），根据玩家当前Build动态展示

**风险：单局太短/太长**
- **预案**：设置两个关键数值旋钮（战斗冷却时间 + 境界节点阈值），可快速调节单局长度
